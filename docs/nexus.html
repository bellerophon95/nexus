01 — Overview

What NEXUS Does

NEXUS is a domain-agnostic research intelligence platform that ingests heterogeneous documents, builds a semantically-rich knowledge base, and answers complex multi-hop questions through coordinated specialist agents. It is designed to demonstrate every layer of a production AI system — from chunking strategy to deployment observability.

⚡

Adaptive Query Routing

A query complexity classifier dynamically routes between direct LLM response, single-step RAG, and multi-hop agentic retrieval — reducing cost by up to 4× on simple queries.

🔀

Multi-Agent Orchestration

A LangGraph supervisor coordinates specialist agents (Researcher, Analyst, Validator) through a stateful directed graph with conditional edges, human-in-the-loop checkpoints, and cycle-based reflection.

🔍

Hybrid Retrieval Pipeline

Dense vector search (sentence-transformers) + sparse BM25 via reciprocal rank fusion, followed by cross-encoder reranking and Self-RAG relevance gating before generation.

📊

Evaluation & Evals

RAGAS framework with faithfulness, answer relevance, context precision/recall metrics. LLM-as-Judge scoring. Automated regression testing on golden datasets before every deploy.

🛡️

Guardrails & Safety

Multi-layer defense: input PII anonymization (Presidio), prompt injection detection (LLM Guard), output hallucination scoring, topic restriction, and toxicity filtering.

📡

Full Observability Stack

Langfuse traces every agent step, tool call, retrieval, and LLM invocation. Cost tracking, latency budgets, quality dashboards, and automated alerts on drift detection.

Semantic Chunking Strategy

Fixed-size chunking achieves ~0.49 faithfulness. Semantic chunking achieves ~0.81. The system computes sentence-level embeddings and splits at cosine similarity valleys, preserving semantic coherence. Parent-child document hierarchy allows retrieval of surrounding context when a specific chunk is matched. Table-aware splitting detects tabular structures and keeps them intact rather than fragmenting rows across chunks.

Python class SemanticChunker: def __init__(self, embed_model, threshold=0.3, min_chunk=100, max_chunk=512): self.embed_model = embed_model self.breakpoint_threshold = threshold # cosine distance threshold def chunk(self, document: ParsedDocument) -> list[Chunk]: sentences = self.split_sentences(document.text) embeddings = self.embed_model.encode(sentences, batch_size=64) distances = cosine_distances(embeddings[:-1], embeddings[1:]) breakpoints = distances > self.percentile_threshold(distances) chunks = self.merge_at_breakpoints(sentences, breakpoints) # Attach parent doc reference + context header for each chunk return [ Chunk(text=c, parent_id=document.id, header=document.title) for c in chunks ]
02

Hybrid Retrieval with Cross-Encoder Reranking

The retrieval layer combines dense and sparse search through Reciprocal Rank Fusion, then applies a cross-encoder reranker for precision. A Self-RAG relevance gate decides whether retrieved context is sufficient or needs a re-retrieval cycle.

  • Dense search: all-MiniLM-L6-v2 embeddings in Qdrant with HNSW index, cosine similarity, top-50
  • Sparse search: BM25 via rank_bm25 library on tokenized corpus stored in Supabase, top-50
  • Fusion: Reciprocal Rank Fusion (k=60) merges both result sets into a unified top-20
  • Reranking: cross-encoder/ms-marco-MiniLM-L-6-v2 scores (query, passage) pairs, outputs top-5
  • Self-RAG Gate: NLI model scores relevance of each chunk to query. Chunks below 0.7 threshold trigger a query-rewrite and re-retrieval cycle (max 2 retries)
  • Metadata pre-filtering: Date range, document type, source filters applied before vector search for efficiency
Python async def hybrid_retrieve(query: str, filters: MetadataFilters) -> list[RankedChunk]: # Parallel dense + sparse retrieval dense_task = qdrant_search(query_embedding, filters, top_k=50) sparse_task = bm25_search(query_tokens, top_k=50) dense_results, sparse_results = await asyncio.gather(dense_task, sparse_task) # Reciprocal Rank Fusion fused = reciprocal_rank_fusion(dense_results, sparse_results, k=60) # Cross-encoder reranking reranked = cross_encoder.rank(query, fused[:20]) # Self-RAG relevance gate relevant = [c for c in reranked[:5] if c.relevance_score >= 0.7] if not relevant: rewritten = await rewrite_query(query) return await hybrid_retrieve(rewritten, filters) # retry once return relevant
03

Adaptive Query Router

Not every query needs the full agentic pipeline. A lightweight LLM-based classifier (distilled to a fine-tuned DeBERTa-v3-small) routes queries into three tiers, reducing cost by up to 4× on simple queries while preserving quality on complex ones.

  • Tier 1 — Direct LLM: Simple greetings, definitions, opinions. Bypasses retrieval entirely. Cost: ~0.001¢/query
  • Tier 2 — Single-Step RAG: Factoid lookups, specific data points. One retrieval + generation pass. Cost: ~0.02¢/query
  • Tier 3 — Agentic Multi-Hop: Comparative analysis, multi-source synthesis, reasoning chains. Full agent loop. Cost: ~0.08¢/query
Python class QueryRouter: """Routes queries by complexity. Trained on 2K labeled examples.""" def classify(self, query: str) -> Literal["direct", "rag", "agentic"]: features = self.extract_features(query) # length, entity count, question type return self.classifier.predict(features) def route(self, query: str, state: GraphState) -> str: tier = self.classify(query) match tier: case "direct": return "generate_node" case "rag": return "retrieve_node" case "agentic": return "supervisor_node"

Multi-Agent Orchestration (LangGraph)

The agent layer implements a Supervisor/Worker pattern via LangGraph's StateGraph. The Supervisor delegates to three specialist agents, each with access to a shared tool registry. Conditional edges enable a reflection loop where the Validator can reject an Analyst's output and trigger re-processing.

  • Supervisor: Routes tasks, manages agent turns, enforces max iterations (3 cycles), aggregates final output
  • Researcher Agent: Has access to vector_search, web_search, and sql_query tools. Retrieves and organizes source material
  • Analyst Agent: Synthesizes retrieved context into structured analysis. Performs multi-hop reasoning across documents
  • Validator Agent: Runs NLI-based fact-checking against retrieved sources. Scores hallucination risk. Can trigger re-analysis via reflection edge
  • Shared State: ACP-style message bus in LangGraph state. Pydantic-validated messages with sender, content, confidence_score, and source_citations
  • Persistence: SQLite checkpoint allows conversation resumption and time-travel debugging
Python · LangGraph from langgraph.graph import StateGraph, END graph = StateGraph(NexusState) graph.add_node("router", query_router) graph.add_node("supervisor", supervisor_agent) graph.add_node("researcher", researcher_agent) graph.add_node("analyst", analyst_agent) graph.add_node("validator", validator_agent) graph.add_node("generate", generation_node) # Adaptive routing from entry graph.add_conditional_edges("router", route_by_complexity) # Supervisor delegates to specialist agents graph.add_conditional_edges("supervisor", delegate_to_agent) # Reflection loop: validator can reject and re-trigger analyst graph.add_conditional_edges("validator", { "approved": "generate", "rejected": "analyst", # reflection cycle "max_retries": "generate", # bail with best-effort }) graph.set_entry_point("router") app = graph.compile(checkpointer=SqliteSaver("nexus.db"))
05

Multi-Layer Guardrails

Defense-in-depth across input and output boundaries. Guardrails run at the application layer and are complementary to model-level safety training. Every guardrail invocation is traced in Langfuse for monitoring.

Input Guardrails

  • Prompt Injection: LLM Guard scanner classifies user input for injection patterns. Blocks and logs attempts
  • PII Anonymization: Microsoft Presidio detects and masks sensitive data (SSN, email, phone) before LLM context
  • Topic Restriction: Zero-shot classifier gates off-topic or restricted domain queries
  • Token Budget: Hard limit on input tokens prevents abuse. Rate limiting per session

Output Guardrails

  • Hallucination Score: NLI model computes entailment between generated answer and retrieved context
  • Citation Verification: Every claim must trace to a retrieved chunk. Uncited claims are flagged
  • Toxicity Filter: Post-generation toxicity and bias screening via LLM Guard output scanners
  • PII Leak Detection: Re-scan output for accidental PII leakage from training data or context

08

Semantic Caching Layer

Production RAG systems see significant query repetition. Rather than exact-match caching, NEXUS implements semantic caching: incoming queries are embedded and compared against cached query embeddings. If cosine similarity exceeds 0.95, the cached response is returned without re-executing the pipeline. This eliminates redundant LLM calls and reduces p95 latency for repeated queries from ~2s to ~50ms.

  • Storage: Upstash Redis with JSON values (response + metadata + TTL)
  • Similarity: Cosine distance on query embeddings with 0.95 threshold
  • TTL: 24h default, configurable per document source (shorter for real-time data)
  • Invalidation: Cache entries are purged on document re-ingestion via pub/sub

06 — Future Improvements

Roadmap

NEXUS is designed with extensibility in mind. Each improvement below adds a new dimension of production sophistication.

GraphRAG Integration

Build a knowledge graph layer (Neo4j/Memgraph) alongside vector search for entity-relationship reasoning. Enables questions like "What connects X to Y?"

Fine-Tuned Embeddings

LoRA fine-tune sentence-transformers on domain-specific data using PEFT. Expected 10-20% relevance improvement on domain queries.

Multimodal RAG

Extend ingestion to process images, charts, and diagrams using ColPali/ColQwen multi-vector late interaction for native visual retrieval.

A/B Prompt Testing

Integrate prompt versioning with Langfuse to A/B test different system prompts and measure impact on RAGAS metrics in production.

MCP Server

Expose NEXUS as a Model Context Protocol server, allowing any MCP-compatible agent to use it as a research tool with standardized context exchange.

User Feedback Loop

Thumbs up/down and correction signals feed back into the golden evaluation dataset. RLHF-lite approach to continuous quality improvement.
